## 🛠️ 步骤

### 1. 云跑 seed
- ssh 进云 → docker exec qmwx-server 跑 prisma seed 脚本
- 预期：AppConfig 3 行写入（feature_flags / member_levels / points_rules）
- seed.ts 是 upsert 幂等，重复跑不爆

### 2. 验证 seed
- `docker exec qmwx-pg psql -U postgres -d qmwx -c "SELECT * FROM \"AppConfig\";"`

### 3. 改 USE_MOCK_DATA = false
- `apps/miniprogram/miniprogram/services/api.ts`
- 保留 mock 函数 + 顶部说明（未来可一键启用）

### 4. 验证首页真链路
- 微信开发者工具 → 重新加载 → 扫码
- 看 sport.myStats 走真 API，云端日志有新 POST
- （首页 userStats 数据可能全是 0 — Checkin 表空，正常）

### 5. 写 e2e prod-smoke
- 新建 `apps/server/tests/e2e/prod-smoke.e2e.test.ts`
- 默认 skip（PROD_SMOKE=1 启用）
- 测：GET /health 200、POST /api/user login 走 code2Session

### 6. CI 加 PROD_SMOKE gate
- `.github/workflows/ci.yml` 加 1 个 job
- 仅 main 分支跑（含手工 dispatch）
