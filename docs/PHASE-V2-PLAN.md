## 0. 文档目的

V2 = **大健康生活方式平台**的真正"健康"部分：

| 方向 | 来源 | 关键能力 |
| --- | --- | --- |
| **设备** | 06 §1+§2 | 蓝牙心率 / 微信运动 / 华为 / 佳明 / 小米 / 荣耀 自动同步打卡 |
| **饮食** | 07 | 菜谱 / 营养查询 / 拍照识菜（百度 AI） |
| **律动** | 08 | 与公司内部运营平台打通（用户 / 订单 / 积分 / 内容下发） |

**核心闭环**（V1 缺的两条腿）：
```
运动（已实现） + 饮食（V2 加） + 设备自动同步（V2 加）
   ↓
   真正的"大健康生活方式平台"：运动消耗 + 饮食摄入 对比，
   配合律动后台运营内容，形成日活 + 留存。
```

## 4. V2 风险登记

| 风险 | 影响 | 缓解 |
| --- | --- | --- |
| 厂商审核延期 1-2 周 | 阻塞 V2 进度 | 提前申请，与开发并行；期间用微信运动 + weRun 兜底 |
| 蓝牙兼容性（部分安卓） | 设备同步部分机型失败 | 准备 3+ 真机回归清单；不通过走手动打卡 |
| 律动 API 变更 | 数据同步失败 | 契约文档 + `/v1` 路径版本化；契约变更走正式 PR 流程 |
| 第三方 API 停服 | 营养查询断流 | 缓存层 + 适配器模式，切换供应商只改适配器 |
| 拍照识菜合规 | 食安审核风险 | 营养建议类文案加"仅供参考，不构成医疗建议" |
| 食品类目资质 | 商城审核 | V1 提审前必须办；V2 内容运营要合规 |

## 5. V2 不在范围（明确划出去）

- ❌ **智能体 / AI 视频 / 抖音发布**（01 §P2-8）— 监管风险，搁置
- ❌ **直播 / 短视频** — 平台资质复杂
- ❌ **iOS App** — 二期再开
- ❌ **海外版**（Fitbit / Strava / Apple Health）— 国内厂商未做透前不开

## 6. V2 干完的样貌

**对用户**：
- 自动同步手表/手环的运动数据（不用每次手动打卡）
- 拍照记饮食 → 看到"今日消耗 vs 摄入"对比
- 食谱/营养查询就在 app 内

**对运营**：
- 律动后台统一管内容（菜谱 / 课程 / 活动 / 商品）
- 不用登录多个后台

**对老板**：
- 数据闭环：用户在 app 的一切行为 → 自动同步到律动
- 可基于数据做精细化运营（积分、商品推荐、活动推送）
