# docs/ — 文档

> 📍 面包屑：`QM-WX/` → [`根 CLAUDE.md`](../CLAUDE.md) → **docs/**（这里）

---

## 🎯 职责

存放**人类可读的文档**：
- 架构决策记录（ADR）
- 接口契约
- 业务说明
- 部署 / 运维手册
- 任何不属于代码注释的"成段叙述"

> ❌ **不要**把以下内容塞进 `docs/`：
> - API 自动生成的参考（用工具生成，放 `dist/` 或专门站点）
> - 临时笔记（用 Notion / Obsidian 等外部工具）
> - 截图二进制堆砌（建专门的 `docs/assets/` 目录管理）

---

## 📂 建议子结构

```
docs/
├── architecture/  # 架构图、ADR、技术选型
├── api/           # 接口契约（手动维护的、给前端看的）
├── ops/           # 部署 / 监控 / 故障排查
├── business/      # 业务说明（给非技术读者）
└── assets/        # 图片 / 图表
```

> ⚠️ **YAGNI 提醒**：没有内容前别建空目录。需要时再创建。

---

## 📝 写作规范

- **文件名**：`kebab-case.md`（如 `user-authentication-flow.md`）
- **语言**：中文（与服务对象保持一致）
- **代码示例**：必须**能跑**，禁止伪代码占位
- **图表**：优先 Mermaid（GitHub 原生支持），其次 SVG
- **更新日期**：每篇顶部加 `> 最后更新：YYYY-MM-DD`

---

## 🔗 引用约定

- 引用其他文档：相对路径，如 `[架构图](architecture/overview.md)`
- 引用代码：路径 + 行号，如 `src/api/user.ts:42`
- 引用 issue / PR：完整 URL
