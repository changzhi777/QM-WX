# 3D肌肉人 · 运动健康指导功能 交互流程设计

**文档目的**：明确"运动健康指导"功能（原"康复"模块）的交互逻辑与合规边界，供前端开发与产品排期对齐使用。

**核心原则**：功能实质决定合规风险，不是命名决定。因此本流程从交互设计层面把"系统诊断"改造为"用户自主选择 + 系统提供通用指导库"，同时嵌入不可跳过的免责与红线人群筛查。

---

## 一、整体流程图（文字版）

```
入口：用户打开"运动健康指导"功能
    │
    ▼
【节点1】红线人群筛查（强制，不可跳过）
    │
    ├─ 用户勾选"是"任意一项风险项 → 跳转"建议就医"页面，功能到此中止
    │
    └─ 全部勾选"否" → 进入下一步
    │
    ▼
【节点2】3D肌肉人交互界面
    用户在人体模型上"自主点选"疼痛/不适区域
    （注意：是用户主动选择，不是系统扫描/判断/诊断）
    │
    ▼
【节点3】区域确认 + 即时免责提示（强制展示，不可跳过，不可仅放协议里）
    "以下为该区域常见的家庭运动指导内容，不构成医疗诊断或治疗建议。
     如为外伤、持续加重、伴随肿胀发热，请及时就医。"
    │
    ▼
【节点4】系统输出通用运动指导库内容
    - 该区域常见家庭训练/拉伸/放松方法（预设知识库内容，非实时生成诊断）
    - 明确标注内容来源（对应国家职业标准条目编号，增强可信度且可追溯）
    │
    ▼
【节点5】AI个性化调整（后续与教练/饮食模块联动）
    根据用户历史运动数据、AI解读结果，对指导内容做强度/频次的个性化适配
    （适配的是"训练强度"，不适配"病因判断"）
    │
    ▼
用户执行指导内容 → 数据回流 → 下次AI解读迭代
```

## 二、各节点详细设计要求

### 节点1：红线人群筛查（前置，强制）

**目的**：把不适合使用该功能的用户，在功能入口就筛选出去，而不是让他们先进入功能再中途拦截。

**建议筛查问题（勾选式，简单快速，不做成体检问卷）**：
- 是否有未愈合的骨折或近期手术史？
- 是否处于孕期？
- 是否有心脑血管疾病病史？
- 当前疼痛是否伴随肿胀、发热、麻木或明显加重？

**触发逻辑**：任意一项勾选"是" → 直接跳转"建议就医/专业机构就诊"提示页，不进入后续交互，功能对该用户本次访问关闭（不做永久拉黑，下次可重新筛查）。

**产品设计要求**：这个问题不能做成"可跳过"的浮层，也不能默认勾选"否"，需要用户主动、逐项确认。

---

### 节点2：3D肌肉人交互——用户自主点选，不是系统诊断

**关键设计原则**：整个交互的主导权在用户，不在系统。

- 界面呈现：3D人体模型，用户手动点击/滑动选择疼痛部位
- **系统不做的事**：不问"什么时候开始疼""疼痛性质是什么"这类诊断性问诊（这类问诊属于医疗行为特征）
- **系统只做的事**：确认用户选中的部位，进入下一步

---

### 节点3：即时免责提示——嵌在交互流程里，不是甩进用户协议

**核心要求**：这条提示必须在用户完成点选的**那一刻**弹出，而不是让用户完成一整套流程后才被动看到，更不能只写进冗长的用户协议里等着没人看。

**建议文案（可调整措辞，但核心信息不能少）**：
> "以下内容为该区域常见的家庭运动指导方法，不构成医疗诊断或治疗建议。如为外伤、持续加重、或伴随肿胀发热等症状，请及时就医。"

**交互形式建议**：非全屏强制弹窗（避免用户体验太生硬），但需要用户点击"我已了解"才能继续查看下一步内容，且每次进入新的身体部位都需要重新确认（不是首次同意后永久跳过）。

---

### 节点4：指导内容输出——预设知识库，非实时生成诊断

- 内容来源：对接国家职业标准知识库（康复治疗师、公共营养师、社会体育指导员相关国标条目），已在推进中
- **建议在前端界面标注内容出处**（例如"内容参考：XX国家职业技能标准"），这既增强用户信任度，也在合规上留痕，证明内容有权威依据、非AI凭空生成
- 输出形式：动作图解/短视频/文字步骤，明确标注"建议频次""建议强度"，避免使用"每天必须做XX次才能好"这类医疗承诺式表述

---

### 节点5：AI个性化调整——适配强度，不适配诊断

后续与教练、饮食模块打通时，AI基于用户的运动数据、历史反馈，调整的应该是：
- 训练强度（轻/中/高）
- 训练频次
- 动作难度递进

**不应该做的事**：不应该让AI根据用户反馈的"疼痛变化"重新判断病因或给出诊断性结论（比如"你这个可能是XX损伤"），这类判断应该始终引导用户"如持续不适请就医"，而不是由AI下结论。

---

## 三、给前端开发的核心要点小结

1. 红线人群筛查页面必须是流程中的**强制前置节点**，不可跳过、不可默认同意
2. 3D人体模型的交互是"用户点选"，界面上不要设计成"系统扫描/AI诊断"这类视觉/文案暗示
3. 免责提示需要嵌入在每次选择部位之后，做成轻量但强制确认的交互，而非纯文字协议
4. 指导内容页面需要展示知识库来源标注
5. 以上4点建议本次前端开发排期中一并纳入，避免后续因合规要求返工
